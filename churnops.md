### End-to-End ML Pipeline with Deployment

**Detailed Explanation:**
This project takes a trained customer churn prediction model and builds the full production infrastructure around it. Rather than leaving a model in a Jupyter notebook, you construct every layer a real ML system needs: a scikit-learn preprocessing pipeline serialized alongside the model (fitted once on training data, never refit on serving data), an MLflow experiment tracking setup that logs parameters, metrics, and artifacts across training runs for comparison, a FastAPI service that exposes the model as a typed REST endpoint, a Docker container so the service runs identically in every environment, and a prediction logger with Population Stability Index (PSI) drift detection so you know when the model's input distribution starts shifting away from the training distribution.

The model itself — a scikit-learn Random Forest (with a Logistic Regression baseline) trained on the Telco Customer Churn dataset — is deliberately not the hard part. The hard parts are the engineering layers around it: a ColumnTransformer preprocessing pipeline that handles mixed feature types correctly, a FastAPI dependency injection pattern that loads model artifacts exactly once at application startup rather than once per request, an explicit separation between internal error detail (full tracebacks and column names, kept inside the backend) and client-safe error messages (short, sanitized, safe to return in the response body), and a drift detection layer that computes PSI over a rolling prediction window and surfaces the results via a /metrics endpoint so model health is observable in production.

The final output is a containerized FastAPI application — served via Docker Compose alongside an MLflow tracking server — that accepts a customer feature payload, applies the saved preprocessing pipeline, returns a churn probability and binary prediction, logs every input-output pair to an append-only SQLite log for monitoring, and exposes a drift summary endpoint. Optional extensions include a Streamlit frontend, a GitHub Actions CI/CD pipeline, cloud deployment, and a batch prediction endpoint.

**Necessary Skills:**
*   **ML Model Training & Serialization:** scikit-learn pipelines, ColumnTransformer, joblib serialization, fitting preprocessors on training data only and reusing the fitted artifact at inference time — never re-fitting on serving data.
*   **Experiment Tracking:** MLflow, logging parameters, metrics, and model artifacts per run; comparing baseline and primary model runs in the MLflow UI; saving training-set feature distributions as reference baselines for drift detection.
*   **REST API Development:** FastAPI, Pydantic v2 request and response models with field constraints and Literal types, dependency injection with Depends, singleton model loading at startup, CORS configuration, custom exception handlers.
*   **Containerization:** Docker, multi-stage Dockerfile builds, .dockerignore, docker-compose for orchestrating multiple services (API container + MLflow tracking server), volume-mounting model artifacts to allow updates without rebuilding the image.
*   **Drift Detection & Monitoring:** Population Stability Index (PSI), numerical feature distribution logging, rolling prediction windows, PSI threshold interpretation (< 0.1 stable, 0.1–0.2 moderate change, > 0.2 significant drift).
*   **Software Engineering:** Project layering (data → model → api → monitoring), configuration via pydantic-settings, separation of error_message_internal from error_message_client, append-only prediction logging with sqlite3.
*   **Testing:** pytest, FastAPI TestClient, patching model artifacts with lightweight mock fixtures so tests never depend on real trained model files.
*   **CI/CD (Optional):** GitHub Actions, automated test runs on push, Docker build verification, pip dependency caching.
